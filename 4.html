비주얼 베이직과 c언어의 차이점

비주얼 베이직과 C언어의 차이는 간단히 말해서 C가 더 기초적인 언어입니다.
이해자체는 VB가 더 쉽지만 C가 좀더 기초언어이다보니 C를 먼저 익히고 다른언어를 하면 다른언어를 더 쉽게 배우실수 있습니다.
VB보다 차라리 자바언어를 배우세요 요즘 프로그래밍은 자바로 많이 이루어지고 있습니다.

먼저 아셔야 될것이.. Visual Basic은 Basic의 확장형태 개발툴입니다.
이 베이직이란 언어자체가 탄생한 것이 프로그래밍을 초급자도 쉽게 할 수 있도록 교육목표로 시작되었다는 겁니다.
그래서 이름이 Basic(기초적인)이에요. 그런데 Visual Basic으로 넘어오면서 본래의 취지가 많이 퇴색되었습니다.
문법자체는 다른 언어에 비해 직관적이라 쉬운건 그대론데 여러 컴포넌트를 언어들이 공유하게 되면서 비주얼 베이직만의 쉽다는 장점이 부각되지 못하고 있습니다.그리고 비주얼 베이직은 컴파일러가 아닌 인터프리터 입니다.
요건 인터넷 검색 좀 더 해보시면 아시겠지만덩치큰 프로그램을 만들기에 베이직은 부적합하다는 뜻입니다.
실제로 비주얼 베이직으로 릴리즈 되는 제품은 거의 없으며, 리포트 숙제용, 혹은 알고리즘 테스트용 등으로간단하게 할 수 있는 프로그램으로 많이 쓰입니다. 어차피 Visual Studio 를 구매하면 따라오는거니까요..

아무튼 비주얼 베이직을 학습하고 C언어를 학습하게 되면 프로그램이란것의 흐름을 이해하는 순서에는 도움이 될런지는 몰라도 c같은
파스칼라이크 언어를 배우게 되면 개념을 또 새로 쌓아가야 되는것이 많아요... 정작 c를 공부하고 나면비주얼 베이직은 공부고 나발이고 할 것도 없어요.. 그래서 저는 c나 c++ 혹은 JAVA 를 공부하시길 추천합니다.

C 와 VB 는 기본적으로 언어 입니다. 서울말로 글을 쓰나 부산 사투리로 글을 쓰나 대한민국 사람은 누구나 알아들을 수 있습니다.
이 서울말과 사투리가 c와 VB 그리고 대한민국 사람이 컴퓨터.. 즉 cpu 가 되겠습니다.
표현 방법(문법이)다르고 동작방법이 다릅니다. 이건 컴파일러와 인터프리터의 차이인데... 이건 또 한 바닥 쭉 써야 되니 패스하겠습니다.

프로그램 짜는사람 입장 혹은 사용하는 사람 입장에서는 동작방법이 다른건 크게 중요하진 않습니다.
그리고 c 의 가장 큰 장점은 시스템 프로그램에 용이하다는 겁니다. 쉽게 말하면 컴퓨터의 장치제어 라던지다른 어플리케이션을 간섭(운영체제에서 허용하는 범위에서 데이터를 훔치는걸 후킹이라고합니다.)해서 이런저런 고급 테크닉을 구사하기 용이합니다.
반면 고급언어 일수록 모듈화되고 중간 과정이 숨겨지고 가려져서... 프로그램을 운용하는 사람의 편리성은 높아지지만 자유도가 낮아진다는 단점이 있습니다.

참고로 프로그램은 저급언어 고급언어 라는 표현을 쓰는데, 고급언어라는게 좋은언어라는게 아니고
저급언어라 해서 나쁘다라는 뜻이 아닙니다.하드웨어에 가까운 언어일수록 자유도가 높고, 코드 짜기가 어렵고 편리성은떨어집니다.
이를 저급언어라 하며 고급언어일수록 사용하기 편하지만 자유도가 떨어지고 제한적 프로그래밍을 하게됩니다.
VB는 c 보다 고급언어에 가깝고, c는 vb보다 저급언어쪽이다.. 라고 보시면 됩니다.
